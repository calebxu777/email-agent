This is a high-level architectural summary of your Frontier Email Agent, designed for maximum reliability, accuracy, and enterprise-grade safety. Your design moves away from simple "chat" and into Deterministic Agentic Workflows.1. The Input Layer: Multi-Modal IngestionThe system begins by transforming messy email data into a structured "Source of Truth."Extraction: Uses a dual-track approach. It parses the Cleaned HTML (converted to Markdown) alongside a High-Resolution Screenshot (processed via VLM) to ensure spatial context (tables, buttons, attachments) is preserved.De-noising: Strips CSS/Boilerplate and uses "Zoning" to isolate the new message from signatures and historical reply chains.The Router: A specialized classifier identifies the intent (Tracking, Return, Complaint, Inquiry) and instantiates the corresponding Category-Specific Worksheet.2. The Logic Layer: Worksheet-Based State ManagementInstead of an open-ended conversation, the agent is governed by a Worksheet—a declarative schema that acts as a "Smart Form."Slot-Filling: The LLM maps natural language to specific fields (e.g., order_id, photo_url).Readiness Gate: A deterministic Python check (via Pydantic) ensures no downstream actions occur until the "Definition of Done" for that worksheet is met.Hybrid Control: Python handles the "Hard Rules" (regex, date math), while the LLM handles the "Fuzzy Logic" (extracting intent and entities).3. The Accuracy Layer: Chain-of-Verification (CoVe)To eliminate hallucinations, the system runs an internal audit before any APIs are touched.Factored Execution: The agent generates "Verification Questions" (e.g., "Is this a 10-digit UPS number?") and answers them in a clean context window to prevent self-bias.Consensus Gate: Compares the initial extraction with the verification results. If they disagree, a "Tie-breaker" reasoning model or a Human-in-the-Loop is triggered.Certification: Once verified, the worksheet is marked as CERTIFIED, creating a "Verified Intent" artifact.The Workflow PipelineStageProcessOutcomeIngestHTML + VLM ScreenshotStructured Markdown + Visual ContextRouteIntent ClassificationSpecialized Worksheet InitializationAuditCoVe Cycle (Plan $\rightarrow$ Verify $\rightarrow$ Reconcile)Certified Worksheet (No Hallucinations)ExecuteAPI / RAG / EscalationReal-world data retrieval/actionDraftWorksheet-Grounded GenerationPolicy-compliant email draftSafetyPost-Gen Compliance AuditRejection of "Bad Promises" or nonsense4. The Execution Layer: Downstream ToolsActions are triggered only by Certified States, never by raw LLM guesses.Direct API Calling: Uses the certified worksheet data to hit logistics backends (Shopify, FedEx, etc.).Recommendation Stream: Uses a RAG + Reranker architecture to pull precise product data for inquiries.Human-in-the-Loop (HITL): High-severity complaints or ambiguous return photos are escalated to a human dashboard with a pre-filled "Context Brief."5. The Output Layer: Guarded Response GenerationThe final email is generated under strict "Lane Confinement."Response Templates: The worksheet dictates the permissible vocabulary. If refund_approved is False, the generator is logically barred from mentioning it.Compliance Prosecutor: A final "Lightweight" LLM checks the draft against the worksheet. It flags any "Bad Promises" (e.g., promising a refund that wasn't approved) and triggers a rewrite if needed.
